DATA VERSONING USING DVC

    DVC (Data Version Control) is exactly like Git, but built specifically for massive data files and machine learning models.

  ** tracking of data like ( raw.csv,train.csv.test.csv)  using DVC.if in future any new data added e get to know .pip install DVC
  ** we add files through DVC ..dvc init 
  * .gitignore file inside .dvc ..whcih will help to untrack non needed dvc files 
  *  we have to untrack csv files from git to track them from dvc .... dvc add artiifacts/raw.csv ... new file of form raw.csv.dvc will be added.. 




         DVC handles the heavy lifting: Instead of telling Git to track artifacts/train.csv, you tell DVC to track it.

         the "Claim Ticket": DVC takes your massive CSV file, securely stores it somewhere else (like an Amazon S3 bucket, Google Drive, or a separate local folder), and leaves behind a tiny, lightweight text file in your folder called train.csv.dvc. Think of this .dvc file as a coat-check claim ticket.

         Git tracks the ticket: You then tell Git to track that tiny .dvc claim ticket.

[data_transformation.py + utils.py]

1. Define config to store path for saving the preprocessor (`preprocessor.pkl`).
2. Build a preprocessing pipeline:

   * Numerical → impute (median) + scale
   * Categorical → impute (most frequent) + one-hot encode + scale
3. Read train and test datasets.
4. Split each into input features (X) and target (`math_score`).
5. Fit the pipeline on train data and transform both train and test.
6. Combine transformed features with target into final arrays.
7. Save the fitted preprocessing object for future use.
8. Return processed train data, test data, and saved file path.
